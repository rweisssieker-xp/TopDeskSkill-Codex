## Writing Rules

- Start with audience, purpose, scope, and prerequisites.
- Use task-based sections.
- Include do/don't guidance where mistakes are costly.
- Include ownership, validation, and escalation.
- Mark tenant-specific values clearly.
- Avoid claiming exact TOPdesk tenant fields unless provided by metadata, exports, or API samples.

## Output Patterns

- For a new handbook: produce title, audience, scope, chapters, procedures, checklists, and maintenance notes.
- For updates: preserve existing structure and patch only the requested section.
- For executive docs: emphasize outcomes, metrics, roadmap, risks, and decisions needed.
- For operator/admin docs: emphasize step-by-step workflows and escalation.

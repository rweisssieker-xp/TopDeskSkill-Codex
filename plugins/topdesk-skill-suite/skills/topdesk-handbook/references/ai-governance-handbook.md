# AI/KI Governance Handbook

Audience: product owners, service leads, operators, security, and compliance.

## Principles

- AI is assistive by default.
- Human review is required for customer-visible communication, closure, priority escalation, sensitive tickets, and destructive actions.
- All suggestions must be auditable.
- Permissions apply before retrieval and display.

## Approved Use Cases

- Ticket classification.
- Smart routing.
- Ticket summaries.
- Draft replies with citations.
- Knowledge suggestions.
- Duplicate detection.
- SLA-risk prediction.

## Required Controls

- Confidence score.
- Explanation.
- Source references.
- Prompt/model version.
- Accepted/rejected/edited feedback.
- PII minimization.
- Sensitive category flags.

## Release Gate

- Historical evaluation completed.
- Sensitive false positives reviewed.
- RLS/retrieval permissions tested.
- Operator feedback workflow ready.
- Disable switch documented.

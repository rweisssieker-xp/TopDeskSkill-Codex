# TOPdesk Report Design Patterns

## Page Types

- Executive overview: KPI strip, trend, SLA status, top risks, exceptions.
- Service desk operations: queue matrix, aging buckets, operator group workload, SLA-at-risk list.
- Process owner: category trends, reopen/reassignment, flow bottlenecks, quality signals.
- Application manager: refresh health, data quality, mapping gaps, API/OData extraction status.

## Visual Rules

- Use compact KPI cards only for action-driving metrics.
- Use line charts for trends over time.
- Use bar charts for ranked categories, groups, or branches.
- Use matrices for operational queues where users need scan-and-act detail.
- Avoid pie/donut charts for many categories.
- Keep slicers predictable: date, branch/customer, process, category, priority/status.

## Accessibility

- Use high contrast.
- Do not rely on color alone for severity.
- Keep labels visible and short.
- Avoid dense visuals on overview pages; move details to drill-through.

# TOPdesk Admin Configuration

Use this file for categories, statuses, supporting files, operator groups, SSP forms, templates, and configuration governance.

## Configuration Areas

- Categories and subcategories.
- Status values and lifecycle transitions.
- Priorities, impact, urgency, SLA inputs.
- Operator groups and permissions.
- Branches, persons, departments, locations, suppliers.
- SSP forms and service catalog items.
- Change templates and activity templates.
- Asset templates and dynamic fields.
- Knowledge visibility and review ownership.

## Governance Rules

- Keep category trees shallow and reportable.
- Avoid duplicate categories with different names.
- Define owner for each supporting-file area.
- Version major form/template changes.
- Test Power BI mappings after renaming categories/statuses.
- Validate RLS impact before branch/person changes.

## Configuration Review Checklist

- Are categories understandable to operators and useful for reporting?
- Are required fields aligned with routing/SLA needs?
- Are operator groups current and not over-fragmented?
- Are inactive persons/operators excluded from new assignments?
- Are statuses mapped to open/closed/cancelled groups?
- Are SSP forms short enough for end users?
- Are templates aligned with actual workflows?
